✅ With safeenv

  • safeenv setup

Five commands. That's the whole thing.

safeenv init

Creates a .venv and tells you exactly how to activate it. Running it twice on the same project does nothing — it never overwrites your environment.

safeenv freeze

Walks every .py file using Python's AST parser — no code is ever executed — and writes a clean requirements.txt automatically. Handles 60+ import → PyPI name mismatches.

most used
safeenv setup

The one you run after cloning any project. Creates .venv + installs everything from requirements.txt in a single step.

safeenv doctor

Read-only health check. Tells you which Python version you're on, whether .venv exists, whether requirements.txt is present, and which packages are missing. Modifies nothing.

safeenv fix

Acts on what doctor found. Creates the venv if missing, installs missing packages. Think of it as auto-repair for your environment.
